Plan d'adressage

Serveur Web

Le serveur web est un serveur qui permet de gérer les pages web de l'application.
Son adressage sur le LAN est 10.0.0.1:8080.

PFSence

PFSence est le serveur pare-feu qui permet de gérer les connexions entrantes et sortantes.
Son adressage est sur le LAN est 10.0.0.6:8080.
Son adressage sur le WAN est 192.168.0.10:8080.

Serveur de backups

Le serveur de backups est un serveur qui permet de sauvegarder les données de l'application.
Les données sont sauvegardées tous les jours.
Son adressage sur le LAN est 10.0.0.2:8080.

Client Linux

Le client Linux est un client qui permet de tester l'application sur Ubuntu.
Son adressage sur le LAN est 10.0.0.3:8080

Client Windows

Le client Windows est un client qui permet de tester l'application sur Windows.
Son adressage sur le LAN est 10.0.0.4:8080

PFSence

Installation

PFSence est un serveur pare-feu qui permet de gérer les connexions entrantes et sortantes.
Il est installé avec le paquet pfSense.

Configuration

La configuration de PFSence est faite dans le fichier /usr/local/etc/pf.conf.

Backup

Installation

Le serveur de backups est un serveur qui permet de sauvegarder les données de l'application.

Automatisation

Les données sont sauvegardées tous les jours grâce à un script bash qui se trouve dans le répertoire/usr/local/bin/. Ce script est appelé backup.sh.
Le script est lancé automatiquement grâce à un cron qui se trouve dans le répertoire /etc/cron.d/.

Nginx

Installation

Nginx est un logiciel permettant de gérer les requêtes HTTP/HTTPS et de publier des pages web.
Il est installé avec le paquet nginx.

Configuration

La configuration de Nginx est faite dans le fichier /etc/nginx/nginx.conf.

Un fichier de configuration /etc/nginx/sites-available/ est créé pour chaque site web.
Un fichier de configuration /etc/nginx/sites-enabled/ est créé pour chaque site web qui doit être publié sur le LAN.

Il existe par défaut un fichier de configuration /etc/nginx/sites-available/default qui est utilisé par notre projet.
