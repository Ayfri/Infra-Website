Cadre du projet

Présentation du projet

Déploiement d’un système d’information et d’un réseau pour une petite entreprise pour l’aider à communiquer et envoyer des données plus facilement. Cette structure permettra aux personnels un système pratique et le site aidera à se documenter sur la nouvelle infrastructure plus rapidement.

L’infrastructure sera composée d’un parefeu PFSense, deux clients (Windows et Linux), un serveur web Linux, un serveur de backup Linux.

L’accès au serveur linux de sauvegarde peut se faire seulement via le réseau interne de la société (soit les deux clients et le serveur web).

L’accès au site web peut se faire du réseau internet ou externe.

Contexte du projet

Le projet a été entrepris dans un cadre universitaire dont le sujet était de créer un réseau local d’une entreprise et mettre en ligne un site web en utilisant divers langages et outils de notre choix (VMWare, PFSense, NginX, HTML, CSS)

Durant la phase de recherche, l'idée de créer un site regroupant les informations nécessaires à la création et l’organisation du projet nous paraissait important. Les utilisateurs pourront donc se documenter plus facilement.

Contraintes du projet

Le projet se devait de respecter quelques points primordiaux. Tout d’abord les informations nécessaires à la création entière du réseau devraient se trouver sur le site. Ces informations devront être concises et précises pour ne pas déstabiliser l’utilisateur.

De plus, en tant que projet dans un cadre universitaire, la durée définie pour livrer le projet est limitée. Nous devons donc avoir un rythme de travail régulier et autonome, ainsi qu’un programme précis.

Backup

Installation

Pour mettre en place la backup, utilisation de rsync.
Remote synchronization ou synchronisation à distance,c’est un logiciel de synchronisation de fichiers.
Il est fréquemment utilisé pour mettre en place des systèmes de sauvegarde distante ou des points de restauration du système (via l'interface Timeshift)
Pour ce faire, il faut deux serveurs (ici le serveur web et la backup).

Configuration

La configuration de la backup sur le serveur backup est assez simple.
Elle consiste en un fichier de configuration /etc/rsyncd.conf qui contient les informations sur le serveur distant et le chemin de sauvegarde.

Test

Sur le serveur web il faut maintenant installer rsync et configurer un site.
Les sites sont installés dans /var/www/html.
Un script est utilisé pour lancer la sauvegarde de tous les sites.

Note :
Le site web est nommé Infra-Website.

Automatisation

Le script de backup va être lancé tous les jours grâce à un crontab.

Après l'exécution du script de backup une fois, nous pouvons voir les fichiers de backup.

Nginx

Installation

Nginx est un logiciel permettant de gérer les requêtes HTTP/HTTPS et de publier des pages web.
Il est installé avec le paquet nginx.

Configuration

La configuration de Nginx est faite dans le fichier /etc/nginx/nginx.conf.

Un fichier de configuration /etc/nginx/sites-available/ est créé pour chaque site web.
Un fichier de configuration /etc/nginx/sites-enabled/ est créé pour chaque site web qui doit être publié sur le LAN.

Il existe par défaut un fichier de configuration /etc/nginx/sites-available/default qui est utilisé par notre projet.
